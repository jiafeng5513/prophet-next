## 二、详细问题分析

### 问题 1：市场分析页面报告渲染效果差

**现状**：`StockAnalysis.vue` 中的 `renderReport()` 函数将后端返回的结构化数据（meta/summary/strategy/details）拼接成 Markdown 字符串，再通过 `markdown-it` 转成 HTML 渲染。

**DSA 对比**：DSA 使用独立 React 组件（ReportOverview, ReportStrategy, ReportNews, ReportDetails）渲染，包含：
- 情绪评分仪表盘（ScoreGauge，0-100 圆环图）
- 实时价格卡片（带涨跌颜色标识）
- 操作建议彩色徽章（买入/持有/卖出）
- 策略点位 4 列卡片网格（带渐变底边框）
- 可折叠的详情面板
- 复制按钮

**Prophet-next 缺失**：
- ✗ 所有内容以纯文本/Markdown 表格呈现，无可视化组件
- ✗ 表格样式不够清晰（缺乏背景色区分、边框样式）
- ✗ 没有仪表盘、徽章、卡片等视觉元素
- ✗ 报告内容在 `max-width: 900px` 容器内居中，全屏时两侧大量空白

---

### 问题 2：聊天(问股)功能 Markdown 渲染简陋

**现状**：`index.js` 中的 `renderMarkdown()` 函数使用正则表达式将 Markdown 转为 HTML。

**问题**：
- 正则解析不完整，嵌套结构和边界情况容易出错
- 不支持 GFM 表格、删除线、任务列表
- 代码块无语法高亮
- 表格样式简陋
- AI 思考过程/工具调用进度在完成后不保留在消息历史中

**DSA 对比**：使用 `react-markdown` + `remark-gfm`，完整支持 GitHub 风格 Markdown。

---

### 问题 3：设置页面混乱

**现状**：`Settings.vue` 硬编码了设置项，布局为单列平铺。

**问题**：
- 分类导航不清晰，所有设置混在一起
- 缺少 DSA 中的多个设置类别（agent 配置、通知渠道、数据源策略、认证设置等）
- 没有字段级别验证和错误提示
- 没有环境配置导入/导出功能
- AI 模型渠道配置功能虽有，但 UI 不够直观

**DSA 对比**：左侧分类导航 + 右侧表单区域，Schema 驱动的动态表单生成，支持条件字段渲染。

---

### 问题 4：端口冲突

**现状**：prophet-next 后端默认使用端口 8000，与 DSA 独立运行时的默认端口相同。

**影响**：无法同时运行两个程序进行对比调试。

**需要修改的位置**：
- `StockAnalysis.vue` — `dsaPort = 8000`
- `index.js` — `dsaPort = 8000`
- `Settings.vue` — 端口输入默认值
- `src/main/index.js` — 后端启动参数
- 后端启动脚本

---

### 问题 5：报告缺少结构化视觉组件

**需要为 Vue 实现的组件**：

| 组件名 | 功能 | 优先级 |
|--------|------|--------|
| ScoreGauge | 情绪评分圆环仪表盘 | 高 |
| PriceCard | 实时价格展示卡（含涨跌色） | 高 |
| AdviceBadge | 操作建议徽章（买入/持有/卖出） | 高 |
| StrategyGrid | 策略点位 4 列卡片 | 高 |
| NewsCard | 新闻资讯卡片列表 | 中 |
| CollapsiblePanel | 可折叠详情面板 | 中 |
| CopyButton | 一键复制按钮 | 低 |

---

### 问题 6：新闻资讯未单独获取

**现状**：新闻内容从 `dashboard.intelligence` 中直接读取字符串嵌入报告。

**DSA 做法**：单独调用 `/api/v1/history/{id}/news` 获取结构化新闻列表（标题、摘要、链接），以卡片形式展示，支持限制数量和刷新。

---

### 问题 7：聊天会话管理 UI 不完善

**现状**：有会话列表侧栏，但功能较基础。

**DSA 对比功能**：
- 会话分组（今日/本周/更早）
- 会话搜索
- 技能选择器（缠论/波浪理论/趋势分析等）
- 快速问题模板
- 发送到通知渠道

---

### 问题 8：代码块无语法高亮

**现状**：代码块仅用 `<pre><code>` 包裹，无语言级别高亮。

**方案**：集成 `highlight.js` 或 `Prism.js`。

---

### 问题 9：设置缺少 Schema 驱动

**现状**：所有设置 UI 硬编码在 Settings.vue 中。

**DSA 做法**：后端 `/api/v1/system/config` 返回配置 Schema，前端动态生成表单，字段类型自动映射（text/number/boolean/select/password）。

---

### 问题 10：组合管理/回测页面视觉效果较简陋

**现状**：功能完整可用，但使用原生 HTML 表格和 CSS 模拟图表。

**DSA 对比**：使用 Recharts 的饼图、柱状图等可视化组件，有更丰富的视觉效果。

**优先级**：低 — 功能不受影响，可后续优化。

- ✅ **Agent 后端**：完整的多 Agent 架构（technical/risk/portfolio/intel/decision）
